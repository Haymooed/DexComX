Quick Start
‚ÑπÔ∏è All commands are owner-only. Use run <script> (or o.run) in Discord to execute.
DEXCOMX

Syntax & Modes

Three execution modes, three interchangeable separators

Separators

Any of these separators can be used interchangeably in any command:

>‚ÄîClassic separator
::‚ÄîScoped separator
=>‚ÄîArrow separator
Classic Mode
‚ÑπÔ∏è Legacy-compatible format. Works with the original DexScript-V3 scripts.
CLASSIC
UPDATE > BALL > France > HEALTH > 100
Scoped Mode
‚ú¶ Use dot-notation to scope to a command class, then chain with any separator.
SCOPED
FILTER.UPDATE :: BALL :: REGIME :: Democracy :: Republic FILTER.VIEW :: BALL :: REGIME :: Democracy FILE.READ :: ./settings.json
Slash-Style Tokens
SLASH-STYLE
/show => BALL => France => HEALTH /rm => BALL => France
Comments
COMMENTS
-- this line will be ignored CREATE > BALL > NewCountry
Batch Execution
‚ö†Ô∏è DexComX accepts batch pastes where each line starts with o.run or run. Markdown code fences are stripped automatically.
BATCH
b.run ```sql CREATE > BALL > A UPDATE > BALL > A > REGIME > Communist UPDATE > BALL > A > HEALTH > 100 UPDATE > BALL > A > ATTACK > 90 ```

Global Commands

Core operations for managing model instances

CREATE
Ôºã Creates a new model instance with required fields populated automatically.
ArgumentTypeDescription
MODELrequiredBALL, REGIME, ECONOMY, or SPECIAL
IDENTIFIERrequiredThe name/identifier for the new instance
SYNTAX
CREATE > MODEL > IDENTIFIER
EXAMPLE
CREATE > BALL > Germany
DELETE
‚ö†Ô∏è Permanently deletes the model instance. Aliases: rm, del
ArgumentTypeDescription
MODELrequiredModel class to target
IDENTIFIERrequiredName of the instance to delete
SYNTAX
DELETE > MODEL > IDENTIFIER
UPDATE
‚úé Updates one attribute. If VALUE is omitted on an image field, the attached file is used. Alias: set
ArgumentTypeDescription
MODELrequiredModel class
IDENTIFIERrequiredInstance name
ATTRIBUTErequiredField name to change
VALUEoptionalNew value. Omit for image fields with file attachment.
SYNTAX
UPDATE > MODEL > IDENTIFIER > ATTRIBUTE > VALUE
VIEW
üëÅ Show one field, or all fields when ATTRIBUTE is omitted. Alias: show
ArgumentTypeDescription
MODELrequiredModel class
IDENTIFIERrequiredInstance name
ATTRIBUTEoptionalSpecific field to show. Omit to see all fields.

VIEW > MODEL > IDENTIFIER > ATTRIBUTE (optional)

üìã Lists all editable fields on a model. Alias: fields

Filter Commands

Bulk operations on matching rows

‚ÑπÔ∏è All filter commands support an optional OPERATOR argument which maps to a Django ORM lookup (e.g. icontains, startswith, lt, gt).
FILTER.UPDATE
SYNTAX
FILTER > UPDATE > MODEL > ATTRIBUTE > OLD_VALUE > NEW_VALUE > OPERATOR (?)
EXAMPLE
FILTER.UPDATE :: BALL :: REGIME :: Democracy :: Republic
FILTER.DELETE
SYNTAX
FILTER > DELETE > MODEL > ATTRIBUTE > VALUE > OPERATOR (?)
EXAMPLE
FILTER.DELETE :: BALL :: REGIME :: Communist
FILTER.VIEW
SYNTAX
FILTER > VIEW > MODEL > ATTRIBUTE > VALUE > OPERATOR (?)
EXAMPLE
FILTER.VIEW :: BALL :: HEALTH :: 100 :: gt

Eval Presets

Save and run Python eval scripts from Discord

CommandSyntaxDescription
EVAL.SAVE EVAL > SAVE > NAME Prompts for eval code and saves it as a preset file
EVAL.REMOVE EVAL > REMOVE > NAME Deletes a saved preset by name
EVAL.LIST EVAL > LIST Lists all saved eval presets
EVAL.RUN EVAL > RUN > NAME Executes a saved preset
‚ö†Ô∏è Preset names are capped at 100 characters. After sending EVAL > SAVE > name, the bot will prompt you to send the eval code within 20 seconds.

File System

Read, write, and manage files on the bot's host

CommandSyntaxDescription
FILE.READ FILE > READ > FILE_PATH Sends the file at the specified path as a Discord attachment
FILE.WRITE FILE > WRITE > FILE_PATH Writes the content of an attached file to the specified path
FILE.CLEAR FILE > CLEAR > FILE_PATH Empties the content of a file without deleting it
FILE.LISTDIR FILE > LISTDIR > FILE_PATH(?) Lists directory contents. Defaults to current directory if omitted. Alias: ls
FILE.DELETE FILE > DELETE > FILE_PATH Deletes a file or entire directory (uses shutil.rmtree for dirs)
EXAMPLES
-- List current directory FILE.ls => ./ -- Read a config file FILE.READ > ./config.json -- Write an attachment to a path FILE.WRITE > ./settings.py -- Clear a log file FILE.CLEAR > ./logs/error.log
