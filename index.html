BallsDex Admin Toolkit

DexComX

Script-driven command suite for BallsDex. Create, update, filter, view, and remove model data directly from Discord — flexible separators, speed aliases, batch execution.

⇄

Multi-Separator

Use >, ::, or => — all interchangeable.

⚡

Speed Aliases

set, show, rm, ls, fields — faster commands.

◈

Filter & Bulk Ops

Mass update, delete, or view rows matching any attribute condition.

▶

Eval Presets

Save, list, and run Python eval scripts from Discord.

▫

File System

Read, write, clear, list, and delete files on the bot host.

⊞

Command Builder

Fill a form, get a ready-to-paste DexComX command instantly.

Installation
◈Add the block below to config/extra.toml in your BallsDex root, then restart your bot.
config/extra.toml
[[ballsdex.packages]] location = "git+https://github.com/Haymooed/DexComX.git" path = "dexscript_app" enabled = true editable = false
1
Open config/extra.toml
Located in the root of your BallsDex installation.
2
Paste the package block above
If [[ballsdex.packages]] already exists, add this as a new block beneath it.
3
Restart your bot
BallsDex will pull and install DexComX automatically on startup.
4
Run your first command
In Discord: run ATTRIBUTES > BALL to verify the extension loaded.
Quick Start
◈All commands are owner-only. Use run <script> (or o.run) in Discord to execute.
DexComX
-- Create a new ball and configure it CREATE > BALL > NewCountry UPDATE > BALL > NewCountry > HEALTH > 100 UPDATE > BALL > NewCountry > ATTACK > 90 UPDATE > BALL > NewCountry > REGIME > Democracy
Scoped Mode
-- Bulk update using filter + scoped mode FILTER.UPDATE :: BALL :: REGIME :: Democracy :: Republic
Aliases
-- Speed aliases set > BALL > France > HEALTH > 105 show :: BALL :: France FILE.ls => ./

Syntax & Modes

// Three execution modes — three interchangeable separators

Separators
>—Classic
::—Scoped
=>—Arrow
Classic Mode
◈Legacy-compatible. Works with original DexScript-V3 scripts.
Classic
UPDATE > BALL > France > HEALTH > 100
Scoped Mode
◈Use dot-notation to scope to a command class, then chain with any separator.
Scoped
FILTER.UPDATE :: BALL :: REGIME :: Democracy :: Republic FILTER.VIEW :: BALL :: REGIME :: Democracy FILE.READ :: ./settings.json
Slash-Style
Slash
/show => BALL => France => HEALTH /rm => BALL => France
Comments
Comments
-- this line will be ignored CREATE > BALL > NewCountry
Batch Execution
⚠DexComX accepts batch pastes starting with o.run or run. Code fences are stripped automatically.
Batch
b.run ```sql CREATE > BALL > A UPDATE > BALL > A > REGIME > Communist UPDATE > BALL > A > HEALTH > 100 UPDATE > BALL > A > ATTACK > 90 ```

Global Commands

// Core operations for managing model instances

CREATE
+Creates a new model instance. Required fields auto-populated with defaults.
ArgumentTypeDescription
MODELrequiredBALL, REGIME, ECONOMY, or SPECIAL
IDENTIFIERrequiredName/identifier for the new instance
Syntax
CREATE > MODEL > IDENTIFIER
Example
CREATE > BALL > Germany
DELETE
⚠Permanently deletes the model instance. Aliases: rm, del
ArgumentTypeDescription
MODELrequiredModel class to target
IDENTIFIERrequiredName of the instance to delete
Syntax
DELETE > MODEL > IDENTIFIER
UPDATE
◈Updates one attribute. If VALUE omitted on image field, attached file is used. Alias: set
ArgumentTypeDescription
MODELrequiredModel class
IDENTIFIERrequiredInstance name
ATTRIBUTErequiredField name to change
VALUEoptionalNew value. Omit for image fields with attachment.
Syntax
UPDATE > MODEL > IDENTIFIER > ATTRIBUTE > VALUE
VIEW
◈Show one field, or all fields when ATTRIBUTE omitted. Alias: show
ArgumentTypeDescription
MODELrequiredModel class
IDENTIFIERrequiredInstance name
ATTRIBUTEoptionalSpecific field. Omit to see all.
Syntax
VIEW > MODEL > IDENTIFIER > ATTRIBUTE (optional)
ATTRIBUTES
◈Lists all editable fields on a model. Alias: fields
ArgumentTypeDescription
MODELrequiredModel class
FILTERoptionalNULL — nullable only. VALID — non-nullable only.
Syntax
ATTRIBUTES > MODEL > NULL|VALID (optional)

Filter Commands

// Bulk operations on matching rows

◈All filter commands support an optional OPERATOR mapping to a Django ORM lookup — e.g. icontains, lt, gt.
FILTER.UPDATE
Syntax
FILTER > UPDATE > MODEL > ATTRIBUTE > OLD_VALUE > NEW_VALUE > OPERATOR (?)
Example
FILTER.UPDATE :: BALL :: REGIME :: Democracy :: Republic
FILTER.DELETE
Syntax
FILTER > DELETE > MODEL > ATTRIBUTE > VALUE > OPERATOR (?)
Example
FILTER.DELETE :: BALL :: REGIME :: Communist
FILTER.VIEW
Syntax
FILTER > VIEW > MODEL > ATTRIBUTE > VALUE > OPERATOR (?)
Example
FILTER.VIEW :: BALL :: HEALTH :: 100 :: gt

Eval Presets

// Save and run Python eval scripts from Discord

CommandSyntaxDescription
EVAL.SAVEEVAL > SAVE > NAMEPrompts for eval code and saves as preset
EVAL.REMOVEEVAL > REMOVE > NAMEDeletes a saved preset by name
EVAL.LISTEVAL > LISTLists all saved eval presets
EVAL.RUNEVAL > RUN > NAMEExecutes a saved preset
⚠Preset names capped at 100 chars. After EVAL > SAVE > name, the bot prompts for eval code — respond within 20 seconds.

File System

// Read, write, and manage files on the bot host

CommandSyntaxDescription
FILE.READFILE > READ > FILE_PATHSends file as a Discord attachment
FILE.WRITEFILE > WRITE > FILE_PATHWrites attached file contents to path
FILE.CLEARFILE > CLEAR > FILE_PATHEmpties a file without deleting it
FILE.LISTDIRFILE > LISTDIR > FILE_PATH(?)Lists directory contents. Alias: ls
FILE.DELETEFILE > DELETE > FILE_PATHDeletes a file or entire directory
Examples
-- List current directory FILE.ls => ./ -- Read a config file FILE.READ > ./config.json -- Write attachment to path FILE.WRITE > ./settings.py -- Clear a log file FILE.CLEAR > ./logs/error.log
